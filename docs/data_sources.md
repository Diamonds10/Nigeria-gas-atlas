|---|---|---|---|---|---|---|
| GOGET country-level dashboard (site counts, production, discoveries, FIDs) | Global Energy Monitor | https://globalenergymonitor.org/projects/global-oil-gas-extraction-tracker | 2026-07-21 | CC-BY 4.0 (stated GEM policy; page footer says "All rights reserved" — confirm with GEM before redistributing) | March 2026 release; national aggregates, discoveries/FIDs 2016–2025 | Country-level only — no field geometry, operator, or per-field reserves |
| GOGET field-level assets (project, status, operator, owner, fuel type, lat/lng) | Global Energy Monitor data, served via GreenInfo Network's public GOGET map viewer | https://greeninfo-network.github.io/global-oil-gas-extraction-tracker/ | 2026-07-21 | CC-BY 4.0 (inferred — same GOGET data GEM licenses elsewhere; page itself states no explicit license, confirm before redistributing) | Snapshot dated 2023-08-02 (per HTTP header); 180 Nigerian fields | Stale relative to the March 2026 dashboard above — status counts won't reconcile exactly (168 vs 204 operating). No reserves or production per field, and no field boundary polygons, only point coordinates. GEM's own current-release field file is gated behind a download form (contact scott.zimmerman@globalenergymonitor.org) — worth revisiting if a more current/authoritative version is needed. On the public map, this dataset is split into three independently toggleable sublayers by `fuel_type`: Oil Fields (33), Gas Fields (2), and Oil & Gas Fields — mixed (145, the majority case — most Nigerian fields produce both). **CAVEAT (verified 2026-07-24):** `fuel_type` is unreliable at the field level for identifying gas production. Checked directly against the raw, untouched GOGET download (not a transformation artifact of this repo's processing) — Soku (hosts one of Nigeria's largest gas processing plants), Bonny (site of the Nigeria LNG terminal), and Gbaran (Shell's flagship non-associated gas development, Gbaran-Ubie) are all labeled `oil` in this source, not `gas` or `oil and gas`. The field likely reflects each field's original discovery/development history rather than current production mix. None of these three names appear in the SE4ALL polygon dataset either, so no cross-reference correction was available. Treat the Oil/Gas/Mixed split as indicative of GOGET's own (imperfect) labeling, not as an authoritative account of which Nigerian fields actually produce gas — most Nigerian gas is associated gas from fields this source calls "oil" or "oil and gas" |
| Crude oil production (International Energy Statistics) | EIA | https://www.eia.gov/opendata/browser/international | 2026-07-21 | Open (U.S. government public data) | Annual, 1973–2025 | API key required (free). This EIA v2 route has no natural gas or reserves facets for international data — production (petroleum only) is all it provides |
| Crude oil & natural gas proven reserves | OPEC Annual Statistical Bulletin (4 editions: 2012, 2015, 2020, 2025) | https://www.opec.org/annual-statistical-bulletin.html | 2026-07-21 | Open (public OPEC publication) | Annual, 2007–2024, continuous | Each ASB PDF only tabulates a trailing 5-year window, so 4 editions were combined to build one continuous series (overlap years 2010–2011 cross-checked and consistent). Editions before 2012 return 404 at OPEC's URL pattern, so 2007 is the practical floor without deeper archive digging. Extracted via PDF text parsing (scripts/01_resource/08_process_opec_reserves.py) rather than a structured export — re-verify if OPEC changes PDF layout in future editions |
| Oil/gas field boundary polygons | Nigeria SE4ALL Open Data Portal (GeoNode/GeoServer, dataset "gas_fields") | https://data.nigeriase4all.gov.ng/catalogue/#/dataset/15 | 2026-07-24 | Not stated on the dataset page — public government/donor open data portal; confirm terms before redistributing | 124 polygon records, 57 unique field names (several fields are mapped as multiple separate lease-block polygons) | Extracted via WFS GetFeature (scripts/01_resource/09_download_se4all_gas_fields.py, 10_process_se4all_gas_fields.py). Cross-checked by name against the GOGET field inventory above: 24 of the 57 unique names already exist in GOGET (this dataset upgrades those from point coordinates to real footprint polygons); the other 33 are field names not present in GOGET at all, so they are net-new additions to Resource coverage (see `in_goget_fields` column in the output CSV). 8 of the 124 polygons have no field name at all in the source data (geometry only) — kept in the output but unusable for name-based lookups. No date, operator, or reserves attributes — geometry and a coarse field_type ("Gas_Field" / "Crude Oil and Gas Field") only, so it complements rather than replaces GOGET's status/operator/ownership fields. On the public map, this dataset is split into two independently toggleable sublayers by field_type: Gas Field Boundaries (62, "Gas_Field") and Oil & Gas Field Boundaries — mixed (62, "Crude Oil and Gas Field" plus 2 unspecified folded in here rather than guessed as pure oil or gas). There is no pure-oil polygon category in this source at all — every non-gas-only record is mixed |
| **Individual wells — not obtained (documented gap)** | Investigated: OilMapNG.com, Nigeria Online Wells Portal (Lynx Information Systems/Bilview Energy/NUPRC), NUPRC National Data Repository (ndr.nascp.gov.ng), OpenStreetMap | — | 2026-07-24 | n/a | n/a — no usable dataset obtained | None of the four candidate sources yielded usable well-level data: (1) OilMapNG.com's Oil Blocks/Fields/Wells/Pipelines/Infrastructure tables are all empty (header rows only, no records) and its "map" is a single static, now-404 JPEG image overlay, not vector data — the site appears abandoned. (2) Nigeria Online Wells Portal (nigeriaoilwelldata.com), which claims 6,000+ Niger Delta wells, returns "Geoviewer service nga-oilwelldata does not exist" (confirmed independently by both automated check and live user attempt on 2026-07-24) — the underlying Lynx GeoViewer service is down or discontinued, and Lynx's own site still links to the same broken URL. Even if restored, it is a commercial data broker (well logs sold at $1,500/wellbore) requiring free account signup, so bulk extraction would raise ToS questions. (3) NUPRC's own National Data Repository requires an authenticated/SSO login (likely restricted to licensed operators), not public. (4) OpenStreetMap has only 2 nodes tagged `man_made=petroleum_well` nationally — negligible. Field-level data (GOGET + SE4ALL polygons above) remains the finest spatial resolution available in Resource; revisit if the Lynx portal is restored or NUPRC opens public NDR access |

| Gas transmission pipelines & LNG terminals (GGIT) | Global Energy Monitor, via GreenInfo Network's public map viewer | https://greeninfo-network.github.io/global-gas-infrastructure-tracker/ | 2026-07-21 | CC-BY 4.0 (inferred — same GEM data policy as GOGET; not explicitly restated on the GreenInfo mirror, confirm before redistributing) | Current; 24 Nigerian gas pipelines, 24 LNG terminal train records (route/point geometry) | Multi-country projects (e.g. West African Gas Pipeline, Trans-Sahara Gas Pipeline) are included whenever Nigeria is any one of the listed countries — not exclusively Nigerian assets |
| Oil & NGL transmission pipelines (GOIT) | Global Energy Monitor, via GreenInfo Network's public map viewer | https://greeninfo-network.github.io/global-oil-infrastructure-tracker/ | 2026-07-21 | CC-BY 4.0 (inferred, same caveat as above) | Current; 15 Nigerian oil/NGL pipelines (route geometry) | No refineries or storage terminals — this tracker is pipelines only |
| Oil- and gas-fired power plants (GOGPT) | Global Energy Monitor, via GreenInfo Network's public map viewer | https://greeninfo-network.github.io/global-gas-plant-tracker/ | 2026-07-21 | CC-BY 4.0 (inferred, same caveat as above) | Current; 193 unit-level records across ~60 Nigerian power stations, with lat/lng | Stored here as the physical-asset record; also relevant to Layer 4 (Demand) since power generation is gas demand — Layer 4 processing reads this same processed CSV rather than duplicating the download |
| Oil refineries | Manually compiled from public reporting (NNPC/BPE disclosures, company sites, Global Energy Observatory, news coverage) — this study | — | 2026-07-21 | Derived/self-collected | 4 major refineries: Port Harcourt, Warri, Kaduna, Dangote (~1.1M bpd combined nameplate) | GEM has no open refinery tracker, unlike its extraction/pipeline/power-plant trackers. Five known modular refineries (Waltersmith, OPAC, Duport, Edo Refinery, Aradel; combined ~30,000 bpd) are *not* included — no verified coordinates were found for them; only town/LGA-level descriptions exist in public sources |
| Oil/gas point infrastructure (compressor stations, gas plants, FPSOs, flow stations, terminals) | Nigeria SE4ALL Open Data Portal (GeoNode/GeoServer, dataset "gas_infrastructure") | https://data.nigeriase4all.gov.ng/catalogue/#/dataset/16 | 2026-07-24 | Not stated on the dataset page — public government/donor open data portal; confirm terms before redistributing | 105 point records after cleaning (112 raw, 7 unnamed dropped), spanning 18 asset-type labels: 30 compressor stations, 15 FPSOs (floating production/storage/offloading vessels — a genuinely new asset type not tracked anywhere else in this atlas), 13 gas plants, 9 LNG plants, 7 export terminals, 7 AGG plants, 6 tanker terminals, 6 NAG plants, 4 refineries, plus smaller categories (gas gathering/injection/compression plants, condensate flow stations, nodes, free trade zones) | Extracted via WFS GetFeature (scripts/02_infrastructure/07_download_se4all_gas_infrastructure.py, 08_process_se4all_gas_infrastructure.py). 7 of 112 records had no name at all and were dropped (geometry/type only, unusable for identification). Status, company, and design-capacity fields are inconsistently populated (roughly half are null) and the source "operating" field mixes capacity numbers, ranges, and text ("down") inconsistently — kept as-is but not reliable for filtering. 7 records were manually identified as likely duplicates of facilities already in this layer's refineries/GGIT LNG terminal records (flagged via `possible_duplicate_of` column, e.g. "Bonny NLNG" = Nigeria LNG Terminal, "OK LNG" = Olokola LNG Terminal) — exclude these when mapping to avoid double-counting. A few LNG-Plant-typed records ("Obigbo Node", "Ukanakafu Pigging Station") look like mislabeled pipeline infrastructure rather than true LNG plants — treat the `type` field as indicative, not authoritative. No cross-reference was possible against the GGIT pipeline/power-plant trackers beyond the manual refinery/LNG check above, since this dataset has no stable external ID |

| Seaports & oil/gas terminals (World Port Index) | NGA (U.S. National Geospatial-Intelligence Agency), via HDX mirror | https://data.humdata.org/dataset/world-port-index | 2026-07-23 | Open (public U.S. government data) | 25 Nigerian ports/terminals, incl. Lagos, Port Harcourt, Calabar, Warri, Onne, plus numerous offshore oil terminals (Bonny, Forcados, Escravos, Qua Iboe, etc.) | Underlying file is dated 2017 — port locations are stable but facility-level attributes (cranes, rail service, depths) may be outdated. Overlaps conceptually with the LNG terminal data in Layer 2 (e.g. Bonny) — not deduplicated across layers |

Overpass queries were run against Nigeria's OSM boundary (relation 192787 / area 3600192787) rather than a bounding box, to avoid picking up slivers of neighboring countries. Public Overpass instances are shared, rate-limited infrastructure — the download script retries across three mirrors (overpass-api.de, overpass.kumi.systems, overpass.openstreetmap.fr) with backoff, since single-mirror 406/429/504 errors were common during collection.
